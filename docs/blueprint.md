# **App Name**: MeetEase

## Core Features:

- User Authentication: Secure user registration and login functionality using email and password, essential for accessing scheduling features.
- Schedule Meeting Form: A user-friendly web form for clients to input meeting details (name, mobile, description), select an available time slot, and upload payment proof.
- Client Meeting History: A dedicated tab allowing logged-in users to view a chronological history of their previously scheduled and upcoming meetings.
- Admin Scheduling & Notification Dashboard: An administrative interface for viewing new meeting requests, verifying payment proofs, confirming or rejecting bookings, triggering client confirmation emails, and sending Firebase notifications to the admin.
- Available Slot Management: A system to define and display available meeting slots, with real-time updates as slots are booked or managed by the admin.
- AI Meeting Description Tool: An AI tool embedded within the scheduling form to assist users in generating clear, concise, and professional meeting descriptions based on keywords or brief inputs.

## Style Guidelines:

- Light color scheme, projecting a professional, organized, and clean aesthetic suitable for a scheduling application.
- Primary color: A deep, sophisticated blue (#3333CC) to signify trust and efficiency, serving as a prominent color for interactive elements and branding.
- Background color: A very light, desaturated blue (#F0F0FB) providing a calm and unobtrusive canvas for content, maintaining visual comfort.
- Accent color: A rich purple (#4F007A) for key calls-to-action, highlights, and status indicators, offering an elegant contrast to the primary blue.
- Headline font: 'Space Grotesk' (sans-serif) to give headlines a modern, slightly tech-informed and clear presence.
- Body font: 'Inter' (sans-serif) for all body text, forms, and descriptions, chosen for its excellent readability and neutral, objective appearance.
- Use a consistent set of clean, simple line icons, ensuring quick recognition and an uncluttered user interface, especially on mobile.
- Fully responsive layout across all devices with a mobile-first approach, featuring intuitive bottom tab navigation for quick access to key sections on smaller screens, embodying PWA best practices.
- Subtle and functional animations for user feedback, such as form submissions, loading states, and tab transitions, enhancing the sense of responsiveness without distracting the user.
